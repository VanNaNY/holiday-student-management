# 假期学生管理系统 · 启动指南与成员分工

> 校园假期「离校 / 留校 / 返校」管理系统 — TYUT 2026-06 课程设计
> 仓库：https://github.com/VanNaNY/holiday-student-management

---

## 一、项目概览

| 端 | 目录 | 技术栈 | 端口 |
|----|------|--------|------|
| 后端 API | `backend/` | Spring Boot 3.3 · Java 17 · Maven · MyBatis-Plus · JWT · H2(dev)/MySQL(prod) | 8080 |
| 移动端 H5 | `frontend/` | Vue 3 · Vite · Vant 4 · Pinia | 5173 |
| PC 管理后台 | `admin/` | Vue 3 · Vite · Element Plus · Pinia | 5174 |

三端通过 REST + JWT 通信，接口统一前缀 `/api`，统一返回体 `{code,msg,data}`。

## 三、启动指南（本地开发）

**开三个终端，按「后端 → 移动端 → 后台」顺序启动。** 移动端/后台依赖后端的 `/api`，所以后端必须先起。

### ① 后端（终端 1，必须先启动）

```bash
# 若 JDK/Maven 装在 ~/.local（免 root 场景），先设置环境变量：
export JAVA_HOME=$HOME/.local/jdk17
export PATH=$JAVA_HOME/bin:$HOME/.local/maven/bin:$PATH

cd backend
mvn spring-boot:run -Dspring-boot.run.profiles=dev
```

- 运行在 **http://localhost:8080**，接口前缀 `/api`
- `dev` profile 使用 H2 内存库，启动时自动执行 `docs/db/schema.sql` + `data.sql` 建表并灌入种子数据
- ⚠️ **H2 内存库每次重启数据重置**（演示用，干净可复现）
- 验证：
  - 健康检查 `http://localhost:8080/api/ping`
  - 接口文档 `http://localhost:8080/api/swagger-ui.html`
  - 数据库控制台 `http://localhost:8080/api/h2-console`（JDBC `jdbc:h2:mem:holiday`，用户 `SA`，空密码）

### ② 移动端（终端 2）

```bash
cd frontend
npm install          # 首次运行需要；之后可跳过
npm run dev
```

- 运行在 **http://localhost:5173**，`/api` 自动代理到后端 8080
- 浏览器按 F12 切到手机视图体验
- **签到定位**：浏览器要求 `localhost` 或 `https` 才允许定位，本地 localhost 正好满足
- **地图（高德）**：复制 `frontend/.env.example` 为 `frontend/.env.local` 并填入自己的 Key 与安全密钥（该文件已被 git 忽略，不会提交）。未配置时地图不显示，但定位与签到逻辑照常工作

```bash
cp frontend/.env.example frontend/.env.local
# 编辑 .env.local：VITE_AMAP_KEY / VITE_AMAP_SECURITY（高德「Web端(JS API)」密钥）
```

### ③ PC 管理后台（终端 3）

```bash
cd admin
npm install          # 首次运行需要；之后可跳过
npm run dev
```

- 运行在 **http://localhost:5174**

## 六、生产部署（简要）

1. MySQL 建库并导入 `docs/db/schema.sql`（可选 `data.sql`）
2. 后端打包运行：`cd backend && mvn clean package -DskipTests`，以 `prod` profile 注入数据库/JWT 环境变量后 `java -jar target/*.jar --spring.profiles.active=prod`
3. 前端/后台分别 `npm run build`，将 `dist/` 交 Nginx 托管并把 `/api` 反代到后端

> 详见根目录 `README.md` 的「生产部署」章节。

## 七、常见问题（FAQ）

| 现象 | 原因 / 解决 |
|------|-------------|
| 后端启动报端口被占用 | 8080 已被占用：`ss -ltnp \| grep 8080` 找到进程并结束，或改 `application.yml` 端口 |
| 前端页面 401 / 接口连不上 | 后端没起或没起好；确认 `http://localhost:8080/api/ping` 正常 |
| 点「定位签到」无反应 | 浏览器拒绝定位：必须在 `localhost` 或 https 下，并允许定位权限 |
| 重启后端后数据没了 | 正常，H2 内存库每次重启重置；需要持久化请切 MySQL（prod profile）|
| 地图不显示 | 未配置 `frontend/.env.local` 的高德 Key；不影响签到逻辑 |
| `git push` 报 GnuTLS/TLS 连接错误 | GitHub 网络抖动，重试即可：`for i in $(seq 1 20); do git push && break; sleep 30; done` |
